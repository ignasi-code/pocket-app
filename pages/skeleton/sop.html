#SkeletonSOP

Fast process for turning source pages into navigable static skeletons at 430x932, with page compare, component compare, composition maps, and verifier checks.

Back to inventory · JSON SOP

Master Flow

  1. Inventory page types and canonical URLs.
  2. Capture source at 430x932.
  3. If full-page capture is suspect, use scroll-by-scroll slices.
  4. Build a static DOM clone from the real rendered page.
  5. Transform the clone into semantic skeleton primitives.
  6. Map sections, components, states, repeated patterns, and links.
  7. Create the component-by-component atlas for the page.
  8. Build the skeleton page with internal navigation.
  9. Create side-by-side page compare.
  10. Create component-by-component compare for risky pages.
  11. Run verifier checks before accepting the output.

Core Rules

  • Real-source gate: source crops must prove they came from the original page.
  • Scroll slices: use slices plus a manifest when full-page screenshots repeat, clip, or miss sections.
  • Capture both sides: the scroll-slice fallback applies to source and skeleton screenshots; hide capture scrollbars or crop them out before atlas generation.
  • Stateful media: animated or evolving heroes remain one component with representative states.
  • Function first: preserve CTAs, options, accordions, ratings, icon rows, bullets, forms, tabs, and selected states.
  • Clone first: create a real static DOM clone before skeletonizing, then debug clone fidelity and skeleton fidelity separately.
  • Primitive skeletonization: do not use one generic rectangle dump; headings, paragraph lines, controls, cards, media, icons, fields, ratings, lists, and rows get separate primitive treatments.
  • Primitive manifest: generated skeletons record counts for text lines, heading lines, controls, cards, icons, and media blocks; implausible counts fail before review.
  • No source text leak: preserve layout without leaking source copy, CTA labels, alt text, aria labels, placeholders, or visible media URLs.
  • Component anatomy contract: before skeletonizing a component, lock the ordered child structure: context/chrome, title or media, repeated units, controls, and footnotes.
  • No invented structures: do not convert source rows into cards, paired media into generic panels, or lists into paragraphs just because the height is similar.
  • No invented borders: borders, outlines, shadows, dividers, and card frames appear only when they exist as structural UI in the source component.
  • Content density gate: cards and panels must preserve source content density: image zones, headings, labels, separators, bullets/checks, price/rating pairs, and repeated rows.
  • Row count gate: repeated semantic units must match count and rhythm; three result rows become three result rows.
  • Component boundary gate: source crops must start and end on component boundaries; do not split one component across atlas rows unless both rows are explicitly labeled as continuations.
  • Sticky chrome suppression: repeated sticky headers or nav bars are detected and removed from component crops unless the row is specifically the header/global chrome component.
  • Uniform repeats: repeated items in one family share one component template.
  • Local gaps: lock important vertical gaps inside cards, banners, CTAs, and option groups.
  • Navigation: page-to-page behavior is baked into the skeleton site.
  • Atlas first: every page type gets component-compare/{page}.html before the standalone skeleton is treated as final.
  • Full-scroll coverage: every atlas must keep adding rows until the source scroll depth is covered top to bottom; broad deep-section rows are allowed when the exact component template is still being learned.
  • Matched skeleton coverage: the skeleton side must use matching named component rows through the full scroll; repeated generic placeholders fail the atlas even when height roughly matches.

Animated Home Hero Note

When a hero animates but remains one layout block, capture it as one stateful media component. Use at least the first state and one later state. Skeletonize the persistent structure: media area, overlays, metric chips/cards, label/status bars, and CTA. Do not split animation frames into unrelated page sections.
